# Инструкция по составлению промтов для сессий-исполнителей

Источник: [Anthropic Prompt Engineering Best Practices](https://platform.claude.com/docs/ru/build-with-claude/prompt-engineering/claude-prompting-best-practices)  
Назначение: руководство для архитектора при написании промтов к задачам PLAN.md.

---

## Базовая структура

XML-теги обязательны: разделяют инструкции, контекст, примеры и ограничения —
Claude разбирает их однозначно, без домыслов.

```xml
<role>
Кто исполнитель, что его задача, какое единственное правило сессии.
</role>

<context>
Почему эта задача существует. Что уже сделано. На что опираться.
</context>

<task>
Что сделать — пошагово, с нумерацией. Включая точный формат вывода.
</task>

<examples>
Конкретные trace / числовые примеры для нетривиальных вычислений.
</examples>

<constraints>
Жёсткие ограничения: размер файла, импорты, типы, что не трогать.
</constraints>

<verification>
Чеклист: исполнитель проверяет каждый пункт перед финальным ответом.
</verification>
```

## Ключевые принципы

### 1. Роль — одним предложением
Одно предложение фокусирует поведение и тон. Укажи роль + единственное правило сессии.

```
Ты — Python-разработчик. Задача: переписать один файл. Никакие другие файлы не трогать.
```

### 2. Контекст и мотивация перед задачей
Claude хорошо обобщает из объяснений — объясняй **почему**, а не только **что**.
Длинный контекст ставь в начало, запрос — в конец (улучшает качество на ~30%).

> ✗ `Не используй циклы`  
> ✓ `Только векторные операции — датафрейм содержит 36 000 строк, Python-цикл даст неприемлемый O(n)-overhead`

### 3. Конкретный формат вывода
Укажи явно: что вернуть, как назвать, какой dtype, какой размер файла.
Без этого исполнители переусложняют и добавляют лишнее.

```
Возвращает pd.DataFrame с двумя колонками: target_30s (Int8 nullable), move_30s (float64).
Целевой размер файла: ≤ 55 строк. Единственный импорт: import pandas as pd.
```

### 4. Few-shot примеры с числовым trace
Для алгоритмических вычислений (rolling, resample, веса) — показывай пошаговое
вычисление на 3–5 элементах. Оборачивай в `<examples>`.

```python
# Пример: rolling(3, min_periods=1).sum() на [1,1,1,1,1]
# Результат:       [1, 2, 3, 3, 3]
# avg_uniqueness:  [1.0, 0.5, 0.333, 0.333, 0.333]
# weights:         [2.0, 1.0, 0.667, 0.667, 0.667]
```

### 5. Чеклист в `<verification>`
5–8 конкретных условий, которые исполнитель проверяет **перед** финальным ответом.
Это самый надёжный способ поймать ошибки типа Int8 vs int8, `>` vs `>=`, n_levels.

```
□ dtype target_30s — "Int8" (заглавная I), не "int8"
□ Строгое > в сравнении, не >=
□ Файл ≤ 55 строк, единственный импорт pandas
```

### 6. Позитивная формулировка
Говори что **делать**, а не чего избегать — Claude следует позитивным инструкциям точнее.

> ✗ `Не используй markdown`  
> ✓ `Пиши ответ в виде текущих абзацев прозы`

### 7. Запрет на домыслы о коде
В `<constraints>` всегда добавляй явный запрет на предположения:

```
Не делай утверждений о коде, который не читал. Прочитай файл перед ответом.
```

### 8. Data contracts для функций
Для задач с нетривиальными типами данных — явно описывай входной и выходной контракт:
индекс (UTC DatetimeIndex), dtype, nullable, что будет NaN и когда.

## Антипаттерны

| Делают | Работает лучше |
|---|---|
| «Будь тщательным» | Конкретный чеклист в `<verification>` |
| «Не делай X» | «Делай Y» (позитивная формулировка) |
| Ограничение без объяснения | Объяснение причины (Claude обобщает из него) |
| «Напиши модуль» | «Файл ≤ N строк, функция X → тип Y» |
| Нет примеров для алгоритма | Числовой trace на 3–5 элементах |
| Большой промт без структуры | XML-секции: роль / контекст / задача / примеры |

---

## Чего не делать в промтах для исполнителей

- **Не давай свободу в выборе подхода** там, где подход зафиксирован — опиши алгоритм явно.
- **Не проси «изучить проект»** без указания конкретных файлов — исполнитель читает лишнее.
- **Не используй капслок и восклицательные знаки** (`КРИТИЧНО: ВЫ ДОЛЖНЫ`) — новые модели
  следуют агрессивному языку буквально и начинают чрезмерно срабатывать на инструменты.
- **Не оставляй размер файла на усмотрение** — без явного лимита код раздувается.
